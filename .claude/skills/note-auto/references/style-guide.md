# 記事スタイル設計ガイド

note記事のスタイル別執筆指針。Phase 1 で選択されたスタイルに応じて
本ガイドの該当セクションを **唯一の根拠** として執筆する。

---

## スタイル一覧

| ID | 表示名 | 用途 | 主な狙い |
|----|--------|------|---------|
| `sales` | 有料記事向け（購買意欲喚起型） | 有料note・LP誘導・商品紹介 | 感情訴求・購買意欲・行動喚起 |
| `info` | 無料記事・読み物（情報提供型） | 無料note・解説・知識共有 | 有益性・読みやすさ・共感 |
| `custom` | その他（自由入力） | カスタム指定 | ユーザー指示に従う |

---

## A. 購買意欲喚起型（`sales`）

### A-1. 文体の数値基準（厳守）

| 指標 | 理想値 | 許容範囲 |
|------|--------|---------|
| 疑問形語尾の比率 | 27.5% | 20〜35% |
| 断定形語尾（です/ます/だ）の比率 | 57.5% | 50〜65% |
| 説明形語尾（のです/なのです） | 17.5% | 10〜25% |
| 短文（1〜15文字）比率 | 48% | 42〜55% |
| 中文（16〜30文字）比率 | 32% | 28〜38% |
| 長文（31文字以上）比率 | 20% | 15〜25% |
| 1文1行の比率 | 91% | 85%以上 |
| 空白行使用率 | 94% | 90%以上 |
| 1段落の行数 | 4行 | 3〜5行 |

### A-2. 構成（5セクション）

```
1. オープニング（0〜10%） — 痛みの言語化＋衝撃数値
2. 共感セクション（10〜25%） — 「こんな悩みありませんか」型の箇条
3. ボディ（25〜75%） — 解決策・証拠・恐怖→希望サイクル5回以上
4. クロージング（75〜95%） — 行動喚起・限定性・後悔の恐怖
5. P.S.（95〜100%） — 最重要ポイントのダメ押し
```

### A-3. 必須要素チェック（執筆中の自己レビュー）

- [ ] 最初の3文以内に **疑問形** を1つ以上
- [ ] 冒頭100字以内に **具体的数値**（端数まで・例: 127.8万円）
- [ ] 読者の苦痛を **身体感覚を含めて** 言語化
- [ ] **恐怖→希望** のサイクルを記事全体で5回以上
- [ ] **箇条書きチェックリスト**（「こんな悩みありませんか」型）を1〜2回
- [ ] **証拠**（数値・事例・第三者の声）を主張1つにつき1つ
- [ ] **対比構造**（Before/After、従来 vs 新方式）を3回以上
- [ ] **時間的希少性** をクロージングで明示（「今だけ」「期限」）
- [ ] **P.S.** で最重要ポイントを再掲

### A-4. 心理トリガーの配置マトリクス

| セクション | 主トリガー | 副トリガー |
|----------|-----------|-----------|
| オープニング | 損失回避・恐怖 | 希望・快楽 |
| ボディ前半 | 損失回避・恐怖 | 社会的証明 |
| ボディ中盤 | 希望・快楽 | 権威・専門性 |
| ボディ後半 | 社会的証明 | 希少性・緊急性 |
| クロージング | 希少性・緊急性 | 損失回避 |
| P.S. | 緊急性 | 恐怖 |

### A-5. キラーワード使用頻度（1000字あたり）

| カテゴリ | 目安回数 | 例 |
|---------|---------|----|
| 金額・数値系 | 8〜12回 | 「月収127.8万円」「1,247名が」 |
| 緊急性系 | 5〜8回 | 「今すぐ」「残りわずか」「期限」 |
| 簡単系 | 4〜6回 | 「誰でも」「シンプル」「自動」 |
| 限定系 | 3〜5回 | 「先着」「二度とない」「特別」 |
| 証拠系 | 3〜5回 | 「実績」「保証」「証明済み」 |

### A-6. 数字の書き方（厳守）

```
❌ 「月収100万円」
✅ 「月収127.8万円」（端数で信憑性UP）

❌ 「3ヶ月で」
✅ 「たった87日で」（具体的）

❌ 「多くの人が」
✅ 「1,247名が」（社会的証明）
```

### A-7. 禁止事項

- 「〜と思います」「〜かもしれません」の多用（自信のなさを露呈）
- 同じ語尾を3回以上連続
- 抽象表現（「すごい」「素晴らしい」）— 必ず数値・具体例で置換
- 嘘・誇大表現（「100%成功」「絶対」）
- 法的にグレーな表現（薬機法・景表法注意）

## B. 情報提供型（`info`）

### B-1. 文体の指針

| 指標 | 目安 |
|------|------|
| 文体 | 個人体験ベース・カジュアル丁寧体 |
| 1文の長さ | 平均25〜35文字 |
| 1段落の行数 | 3〜5行 |
| トーン | 共感的・専門的すぎない |

### B-2. 構成（標準型）

```
1. 冒頭フック（0〜15%） — 個人体験・共感から始める
2. 課題提示（15〜25%） — 何を解決するか
3. 本論（25〜80%） — 段階的な解説・具体例
4. まとめ（80〜95%） — 要点3〜5個と次のアクション
5. （任意）あとがき（95〜100%） — 個人的な感想
```

### B-3. 必須要素

- [ ] 冒頭は **個人体験・共感** から（営業色ゼロ）
- [ ] H2大見出し3〜5個、各H2にH3小見出し2個以上
- [ ] **具体例・数値・出典** をリサーチから3件以上
- [ ] **箇条書き** で読みやすさを担保
- [ ] **引用ブロック** で要点を強調
- [ ] **太字** で重要語を強調
- [ ] 文末に「## まとめ」セクション
- [ ] **アクションを1文** で示す

### B-4. やりすぎ禁止事項

- 過度な煽り（「絶対」「衝撃」「ヤバい」の多用）
- 強い販売色（「今すぐ買え」型）
- 端数価格の連発（情報記事には不要）
- 限定・希少性の過剰演出

---

## C. カスタム型（`custom`）

ユーザーがスタイル選択時に **自由入力** で指定した内容を最優先。
A・B いずれかを近似スタイルとして参考にしつつ、ユーザー指定を上書き反映。

## 自己レビューフロー（必須）

執筆完了時に以下を実行：

1. **語尾比率カウント**: 疑問形・断定形・説明形の比率を実測
2. **文長分布**: 短/中/長の比率を実測
3. **キラーワード密度**: sales 型なら頻度目安と照合
4. **構成チェック**: セクション順・必須要素の有無
5. **NG表現の検出**: 抽象表現・誇大表現・法的グレーゾーン

不適合があれば該当箇所を書き直し、最大3回までリトライ。

## メタデータへの保存

`metadata.json` に以下を必ず保存：

```json
{
  "article_style": "sales | info | custom",
  "style_notes": "（custom時のユーザー指示文）"
}
```
