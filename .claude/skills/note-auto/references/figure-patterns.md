# 図解パターン集（Phase 4-B 用 / Claude Code 直接生成ルート）

`output/_preview/v6_dense` 等の検証で得た「note記事に効く図解の黄金則」を、
Claude Code が PIL で図解を毎回新規描画する際の **設計指針** として整理した。

---

## 共通設計原則（必ず守る）

1. **キャンバスサイズ**: 1280 × 720（landscape固定）
2. **外枠余白**: 上下20-30px、左右20-30px。「キャンバスを埋め尽くす」のが正解
3. **タイトルフォント**: **64〜80pt以上**、極太（Bold）。画面上部1/6に配置
4. **本文フォント**: **40〜52pt以上**、太字または中太字。**細字は使わない**（読めない）
5. **小要素フォント**: ラベル等の最小でも **32pt以上**
6. **背景**: 単色パステル（`#FFF8E7` `#E8F4FD` `#FDF0F0` 等）。グラデ/影/3D禁止
7. **配色**: 1図解につき3-4色まで（背景含む）。アクセントは1色だけ目立たせる
8. **パンチライン帯**: 最下部に画像幅いっぱいの黄色帯（`#FFD700`、高さ12-15%、極太黒文字 **52〜60pt**）

> **重要・スマホ表示前提**: noteはスマホで読まれる。1280×720画像を縦300px幅で表示しても
> 文字が読めるサイズにすること。**「PCで見て丁度いい」は失格**、「スマホで見て大きすぎるくらい」が正解。

### 文字数の上限（厳守）

- タイトル: **20文字以内**（長いと縮小されて読めなくなる）
- 本文1行: **15文字以内**
- パンチライン: **20文字以内**
- 比較表の各セル: **8文字以内**
- ステップ図の説明: **10文字以内**

文字数オーバー時は **絶対にフォント縮小しない**。文言を短くする。

## パターン早見表（記事タイプ→推奨パターン）

| 記事タイプ | サムネ向け | 図解1向け | 図解2向け |
|----------|-----------|----------|----------|
| ノウハウ・解説 | キャッチコピー型 | ステップ図 | ポイント箇条 |
| 〇選・まとめ | 数字強調型 | 4象限 | 比較表 |
| 効率化・改善 | ビフォーアフター型 | フロー図 | 効果数値 |
| ツール比較 | 比較分割型 | 比較表 | チェックリスト |

| 概念説明 | 一語強調型 | ベン図/概念図 | 比喩図解 |

## 各パターン詳細

### A. キャッチコピー型（サムネ向き）
- レイアウト: 中央に巨大タイトル（60-80pt極太）、上下にサブ
- アクセント: 右下に小アイコン（チェック・電球・ロケット等）
- 配色: 背景パステル + アクセント1色
- 用途: ノウハウ・解説記事のサムネ

### B. 数字強調型（〇選・ランキング）
- レイアウト: 左1/3に巨大数字（200-300pt）、右に説明テキスト
- アクセント: 数字を黄/赤で目立たせる
- 用途: 「3つの方法」「5選」系のサムネ

### C. ビフォーアフター型
- レイアウト: 左右2分割、中央に「→」または「VS」
- 配色: 左を寒色（×イメージ）、右を暖色（○イメージ）
- 各エリア上部にラベル（「Before」「After」）、中央に成果数値
- 用途: 効率化・改善・導入効果の図解

### D. ステップ図
- レイアウト: 横並び3-5ステップ、各ステップに番号丸＋短文
- アクセント: 矢印で繋ぐ（細い線、装飾控えめ）
- 用途: 手順説明・プロセス紹介

### E. 4象限マトリクス
- レイアウト: 縦横軸＋4ボックス、各ボックスに見出し＋短文
- 配色: 各象限に異なる薄色背景
- 用途: 比較・分類・選択肢の整理

### F. 比較表
- レイアウト: 2列または3列の表、行に評価軸（速度/価格/品質等）
- 配色: ヘッダー行のみ濃色、本体は白/薄色
- 用途: ツール比較・選択肢評価

### G. ベン図 / 概念図
- レイアウト: 2-3円の重なり、各円にラベル、重なり部分を強調
- 用途: 概念の関係性、独自ポジション

### H. タイムライン
- レイアウト: 横軸の線＋ノード、各ノードに日時＋短文
- 用途: 体験談、変化の経緯

### I. ポイント箇条（マインドマップ風）
- レイアウト: 中央に主題、放射状に3-5項目
- 用途: まとめ図解

### J. 一語強調（概念サムネ）
- レイアウト: 画面中央に1単語（120-200pt極太）、上下にサブ
- 用途: 抽象概念の象徴サムネ

---

## やってはいけないこと

- ❌ グラデーション背景（コーポレ感が出てnote向きではない）
- ❌ 影・3Dエフェクト（古臭い印象）
- ❌ 細いフォント（スマホで読めない）
- ❌ 1図解に5色以上（散らかる）
- ❌ 余白を多く取る（密度を上げる方が読まれる）
- ❌ 画像内テキストに「。」を多用（短く言い切る）
